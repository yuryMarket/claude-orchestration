Также: Python-разработчик, Node.js/TypeScript-разработчик.

## AIDD Workflow (автоматическая оркестрация)

Когда пользователь описывает задачу (фичу, баг, улучшение) — запусти AIDD workflow автоматически.
Протокол оркестрации: @~/.claude/workflow.md

### Режим работы

- **Задача описана** → запусти workflow из workflow.md (определи ticket, начни с Этапа 1)
- **Существующий тикет** → прочитай `docs/.active_ticket`, проверь артефакты, продолжи с незавершённого этапа
- **Нет задачи** → работай в обычном режиме без ограничений AIDD

### Протокол пауз

ОБЯЗАТЕЛЬНО останавливайся после каждого этапа и жди подтверждения пользователя.
Перед правками кода: показать план изменений, дождаться подтверждения.

## Инструменты и Workflow

### Выбор субагентов
- Когда пользователь просит "изучить", "исследовать", "проанализировать" информацию — использовать `deep-researcher` агента, а не только тематически подходящего агента.
- Приоритизировать **намерение** пользователя (исследование vs быстрый ответ), а не тему вопроса.
- При исследовательских задачах — запускать `deep-researcher` параллельно с тематическим агентом (например, `claude-code-guide`).

### Субагенты

**AIDD workflow** (вызываются оркестратором автоматически):

| Субагент | Этап | Роль |
|----------|------|------|
| `analyst` | 1. PRD | Создание/обновление PRD |
| `deep-researcher` | 2. Исследование | Техническое исследование |
| `planner` | 3. Архитектура | Архитектурный план |
| `task-planner` | 4. Задачи | Декомпозиция на атомарные задачи |
| `implementer` | 5. Реализация | Код + тесты (одна задача за вызов) |
| `reviewer` | 6. Ревью | Анализ кода |
| `qa-engineer` | 7. QA | Тесты + QA-отчёт |
| `tech-writer` | 8. Документация | CHANGELOG, README, runbooks |
| `validator` | Служебный | Проверка quality gates |

**Вспомогательные** (вызываются по необходимости):

| Субагент | Когда использовать |
|----------|-------------------|
| `subagent-creator` | Создание новых AGENT.md файлов |
| `claude-code-guide` | Уточнение возможностей/формата Claude Code |

### Общие правила
- Library-first: искать готовую библиотеку перед написанием >20 строк кода
- Conventional commits: `type(scope): описание`
- Всегда запускать тесты после изменений в коде
- Никогда не пропускать quality gates

## Язык

- Все файлы в `~/.claude/` (rules, skills, agents, hooks, документация) создаются на **русском языке**
- Исключения: код, команды, технические термины без устоявшегося перевода
