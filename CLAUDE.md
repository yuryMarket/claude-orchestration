## AIDD Workflow (автоматическая оркестрация)

Когда пользователь описывает задачу (фичу, баг, улучшение) — сначала примени маршрутизацию (см. ~/.claude/rules/routing.md): дай рекомендацию о режиме (AIDD / прямая) и спроси пользователя. Запускай AIDD только после его подтверждения.
Протокол оркестрации: `~/.claude/workflow.md` — читай в начале AIDD-задачи.

### Режим работы

- **Задача описана** → НЕ запускай сразу. Сначала маршрутизация (routing.md): рекомендация + вопрос пользователю. После выбора «AIDD» → workflow.md (определи ticket, Этап 1)
- **Существующий тикет** → резолюция активного тикета: per-session файл `~/.claude/sessions/aidd/<session_id>.json` (primary) → `docs/.active_ticket` (legacy fallback); проверь артефакты, продолжи с незавершённого этапа
- **Нет задачи** → работай в обычном режиме без ограничений AIDD

### Протокол пауз

ОБЯЗАТЕЛЬНО останавливайся после каждого этапа и жди подтверждения пользователя.
Перед правками кода: показать план изменений, дождаться подтверждения.

## Инструменты и Workflow

### Выбор субагентов
- Когда пользователь просит "изучить", "исследовать", "проанализировать" информацию — использовать `deep-researcher` агента, а не только тематически подходящего агента.
- Приоритизировать **намерение** пользователя (исследование vs быстрый ответ), а не тему вопроса.
- При исследовательских задачах — запускать `deep-researcher` параллельно с тематическим агентом (например, `claude-code-guide`).

### Субагенты

Роли агентов AIDD-этапов — в `~/.claude/workflow.md` (таблица этапов); назначение каждого агента — в `description` его AGENT.md (доступно при выборе агента). Дубли здесь не ведём.

### Обязательное использование специализированных агентов

При создании или изменении конфигурационных файлов `~/.claude/` — **всегда** использовать соответствующего специализированного агента, никогда не писать файлы напрямую:

| Что создаётся/изменяется | Агент |
|---|---|
| `~/.claude/agents/*/AGENT.md` | `subagent-creator` |
| `~/.claude/rules/*.md` | `rules-manager` |
| `~/.claude/skills/*/SKILL.md` | `skills-manager` |
| `~/.claude/hooks/*.sh` + регистрация в settings.json | `hooks-manager` |

### Общие правила
- Library-first: искать готовую библиотеку перед написанием >20 строк кода
- Conventional commits: `type(scope): описание`
- Всегда запускать тесты после изменений в коде
- Никогда не пропускать quality gates
 

## Язык

- Все файлы в `~/.claude/` (rules, skills, agents, hooks, документация) создаются на **русском языке**
- Исключения: код, команды, технические термины без устоявшегося перевода
