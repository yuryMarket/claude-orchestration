# AIDD Workflow

Оркестратор AIDD. При задаче — проводи через этапы, вызывая субагентов. Агенты читают артефакты с диска, пишут результат, возвращают ≤20 строк резюме.

## Точка входа

1. Определи ticket ID (спроси или предложи `FEAT-001`)
2. Зафиксируй активный тикет (per-session изоляция):
   - (a) Узнай `session_id`: из строки контекста `AIDD session_id: <S>` (её инжектит `session-start.sh`) ИЛИ `Bash(printf '%s' "$CLAUDE_CODE_SESSION_ID")`.
   - (b) Если `session_id` известен — запиши per-session файл `~/.claude/sessions/aidd/<S>.json` с полями `{ticket, cwd, source:"orchestrator", updated}` (атомарно: `python3 -c` через mkstemp + `os.replace`).
   - (c) Запиши legacy `docs/.active_ticket` УСЛОВНО через helper: `Bash(bash ~/.claude/hooks/_lib/aidd-write-legacy.sh "<cwd>" "<ticket>")`. Helper записывает, только если файл пуст или уже содержит тот же тикет — не перетирает указатель параллельной сессии. Вызывай всегда (если `session_id` неизвестен — helper отработает так же; создаёт `docs/` при необходимости).
3. Начни с Этапа 1

Существующий ticket — резолюция per-session (primary): строка `AIDD session_id: <S>` в контексте → файл `~/.claude/sessions/aidd/<S>.json`; fallback — `docs/.active_ticket`. `session-start.sh` уже инжектил активный тикет в контекст при старте. Проверь артефакты, продолжи с первого незавершённого этапа.

## Протокол пауз (ОБЯЗАТЕЛЬНО)

После каждого этапа:
1. Покажи резюме результата + таблицу прогресса
2. **СТОП** — жди: "продолжай" → следующий этап; feedback → перезапусти; "стоп" → останови.

## Этапы

| # | Агент | Промпт агенту | Quality Gate | Действие |
|---|-------|--------------|--------------|----------|
| 1 | `analyst` | `Ticket: X, Описание: <описание>` | `docs/prd/X.prd.md` существует, нет `[BLOCKING]` | Покажи требования + вопросы. **СТОП** |
| 2 | `deep-researcher` _(авто, если `research_needed: true` в PRD)_ | `Ticket: X` | `docs/research/X.md` существует | Покажи резюме. **СТОП** |
| 3 | `planner` | `Ticket: X` | `docs/plan/X.md` существует, архитектура описана | Покажи решения + риски. После подтверждения: `Status: PLAN_APPROVED`. **СТОП** |
| 4 | `task-planner` | `Ticket: X` | `docs/tasklist/X.md` существует, задачи с acceptance criteria | Покажи задачи + оценку. **СТОП** |
| 5 | _см. ниже_ | — | Все задачи `[x]` | Статистика + тесты. **СТОП** |
| 6 | `reviewer` | `Ticket: X` | Нет BLOCKING в `reports/review/X.md` | Авто-фикс BLOCKING (макс 3 цикла). **СТОП** |
| 7 | `qa-engineer` | `Ticket: X` | `reports/qa/X.md` → `Verdict: RELEASE_READY` | Авто-фикс NEEDS_FIX (макс 2 цикла). **СТОП** |
| 8 | `tech-writer` | `Ticket: X` | `CHANGELOG.md` содержит ticket ID | Покажи обновлённые файлы. **СТОП** |

### Этап 5: Реализация (подробно)

Читай `docs/tasklist/X.md`, находи `- [ ]` задачи.

**Тип задачи:** infra (GCP/K8s ресурсы без изменений кода: gcloud/kubectl delete/describe/list/verify) → `infra-operator`; всё остальное → `implementer`.

**Алгоритм:**
1. Из текущей категории (порядок: Infrastructure → Code → Tests → Monitoring → Docs) выбери независимые задачи
2. Запусти батч: infra → `infra-operator`; code → `implementer` параллельно (макс 3)
   - Промпт infra: `Ticket: X, Выполни ТОЛЬКО: <задача>, Логи: <путь>`
   - Промпт code: `Ticket: X, Реализуй ТОЛЬКО: <задача>. НЕ трогай чужие файлы.`
3. После батча: прочитай tasklist, лог прогресса без СТОП, авто-переход к следующему батчу
4. Один и тот же файл → только последовательно; после code-батча → запусти тесты

**СТОП** когда ВСЕ `[x]`.

## Завершение

После Этапа 8: вызови `validator`, покажи итоговую таблицу, предложи `/commit`, PR или правки.

Показывай после каждого этапа:

| 5 | Реализация   | N/M    | код + тесты                |

## Принципы

- Артефакты на диске — агенты читают их сами
- Оркестратор хранит только резюме (≤20 строк на агента)
- Gates проверяются оркестратором, не агентами
- При неопределённости — СТОП и вопрос пользователю
- Активный тикет изолирован по `session_id` (`~/.claude/sessions/aidd/<S>.json`); legacy `docs/.active_ticket` — fallback. Несколько сессий в одной директории ведут разные тикеты независимо.
