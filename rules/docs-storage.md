# Хранение артефактов и документации

Единое правило для всех агентов: куда сохранять PRD, исследования, планы, тасклисты, расследования и отчёты. Разграничение между проектным и глобальным режимами обязательно.

## Два режима хранения

### Режим 1: Задача в рамках проекта

Если рабочая директория — репозиторий проекта (не `~`), все артефакты сохраняются **относительно корня проекта** в каталоге `.claude/docs/`:

| Подкаталог | Назначение |
|---|---|
| `.claude/docs/prd/` | PRD и требования (AIDD Этап 1) |
| `.claude/docs/research/` | Технические исследования (AIDD Этап 2, deep-researcher) |
| `.claude/docs/plan/` | Архитектурные планы, ADR (AIDD Этап 3) |
| `.claude/docs/tasklist/` | Декомпозиция задач (AIDD Этап 4) |
| `.claude/docs/investigations/` | Расследования: security alerts, инциденты, аудиты |
| `.claude/docs/reports/` | Отчёты ревью, QA, валидации |

### Режим 2: Задача вне проекта

Если рабочая директория — `~` или не определена как проект, все артефакты сохраняются в `~/docs/` с теми же подкаталогами:

- `~/docs/prd/`
- `~/docs/research/`
- `~/docs/plan/`
- `~/docs/tasklist/`
- `~/docs/investigations/`
- `~/docs/reports/`
- `~/docs/quick-lookup/` — **только** для агента `quick-lookup`

## Определение контекста

Рабочая директория считается проектом, если в ней (или в одном из родительских каталогов до `~`) присутствует любой из маркеров:

- `package.json`
- `pyproject.toml`
- `Makefile`
- `go.mod`
- `.git`
- `cdk8s.yaml`

Если ни один маркер не найден или `cwd == ~` — использовать Режим 2 (`~/docs/`).

## Общие правила

- Подкаталоги создаются автоматически при необходимости (`mkdir -p`)
- Именование файлов:
  - AIDD-артефакты: `<ticket>.md` (например, `FEAT-001.md`, `FEAT-001.prd.md`)
  - Глобальные / свободные артефакты: `YYYY-MM-DD-тема-через-дефис.md`
- Файл создаётся **до** возврата ответа вызывающему агенту — не возвращать резюме без записанного артефакта
- Один артефакт — один файл; не смешивать разные типы (PRD и план) в одном файле
- Не сохранять артефакты в корень `.claude/docs/` или `~/docs/` — только в профильные подкаталоги

## Запреты

- Не смешивать режимы: если работа идёт в проекте — не писать в `~/docs/`, и наоборот
- Не создавать альтернативные пути (`notes/`, `tmp/`, `output/`) — только утверждённые подкаталоги
- Не сохранять артефакты вне `.claude/docs/` / `~/docs/` (кроме кода и конфигов, которые живут по своим стандартным путям репозитория)
