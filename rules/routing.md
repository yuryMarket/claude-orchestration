# Маршрутизация задач: AIDD vs прямая сессия

## Главный принцип (критично)

Claude **НИКОГДА не выбирает режим сам на 100%**. На каждую новую задачу Claude:

1. Даёт рекомендацию с кратким обоснованием.
2. Задаёт вопрос пользователю, какой режим запускать.
3. **СТОП** — ждёт решения.

Действие начинается **только после ответа пользователя**. Пользователь либо принимает рекомендацию, либо выбирает другой режим — Claude следует решению без спора.

## Когда протокол срабатывает

Срабатывает:
- На каждую новую постановку задачи (фича / баг / улучшение / операция), включая первую в сессии.

НЕ срабатывает:
- Продолжение уже запущенного режима / тикета в той же сессии.
- Чистые вопросы и исследование без изменения кода — просто отвечаем.
- Тривиальные уточнения к текущей работе.

## Rubric классификации

Три вопроса. **≥2 «да» → рекомендуй AIDD**, иначе → прямая сессия.

| # | Вопрос | Если «нет/да» |
|---|--------|---------------|
| 1 | Знаю ли точно, что и как делать? | Нет → +AIDD |
| 2 | Затронет >2–3 файлов или staging/production? | Да → +AIDD |
| 3 | Будет формальное ревью / аудит / ServiceNow CR? | Да → +AIDD |

Дополнительный контекст: активный `docs/.active_ticket` → возможно продолжение AIDD (учесть при рекомендации).

## Сильные сигналы «прямая сессия»

- Вопрос / исследование без правок.
- Тривиальное изменение: 1 файл, опечатка, bump тега TF-модуля (`?ref=`), одно поле в `image.json`.
- Точечный багфикс с ясной причиной.
- SRE-операция: `kubectl` / `gcloud describe` / `list` / `verify`.
- Механика по готовому образцу: скопировать `.tf` агента, сменить `repo_name`.

## Сильные сигналы «AIDD»

- Новая фича с объёмом (>2–3 файлов / компонентов).
- Размытые требования.
- Staging / production → нужен ServiceNow CR и аудит-trail.
- Незнакомая область (сначала research).
- Работа на несколько сессий.

## Формат рекомендации

Выдавать на каждую задачу:

```
🔀 Рекомендация: <прямая сессия | AIDD>
   <2–3 фактора из rubric, через ·>
   Какой режим запускаем? (прямая / AIDD / уточнить)
— жду твоего решения
```

После формата — **СТОП**.

## После ответа пользователя

- Выбор «прямая» → обычный режим без ограничений AIDD.
- Выбор «AIDD» → читай `~/.claude/workflow.md`, Этап 1 (или продолжение существующего тикета).
- Пользователь выбрал режим, отличный от рекомендации → следуй его решению.
