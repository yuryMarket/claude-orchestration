# Контекстная эффективность

Правила для предотвращения неконтролируемого расхода контекста.

## Не вызывать Skill(update-config) без необходимости

`update-config` — встроенный скилл Claude Code. При вызове `Skill(update-config)` весь его контент загружается в текущий контекст. Встроенная версия содержит полную JSON-схему settings.json (значительный объём токенов).

**Запрещено вызывать `Skill(update-config)` если:**
- Файл `~/.claude/settings.json` уже прочитан в текущей сессии
- Известно конкретно какие строки нужно добавить или изменить

**Вместо этого:** `Read ~/.claude/settings.json` → `Edit` → `Bash(jq -e ... ~/.claude/settings.json)` для валидации.

**Разрешено вызывать если:**
- Задача неоднозначна: пользователь описал желаемое поведение, а не конкретную настройку
- Нужна справка по синтаксису редких полей

## Ограничение verbosity при делегировании агентам

При любом исследовательском делегировании (Explore, deep-researcher, quick-lookup при сборе нескольких фактов) — явно ограничивать ответ:

```
Верни краткий список (не более 30 строк). Без цитат кода, без пояснений каждого пункта.
```

Без явного ограничения агенты возвращают подробные отчёты, которые остаются в контексте оркестратора до конца сессии или компакта.

### Не переопределять output contract агентов

У каждого агента есть встроенный output contract (раздел "Контракт возврата оркестратору" в AGENT.md). Нельзя переопределять его в промпте:

**Запрещено:**
- `"Не сохраняй в файл"` — отменяет механизм file-first, выключает защиту от inline-verbosity
- `"под 50/60 строк"` — завышает лимит выше контрактного (≤20 строк для quick-lookup)
- `"Верни текстом напрямую"` — обходит контракт Статус/Артефакт/Резюме

**Правильный промпт к quick-lookup:**
```
Получи Jira тикет COR1-330. Верни: title, type, status, assignee, суть задачи (2-3 предложения), acceptance criteria.
```
Не добавлять инструкции о формате возврата — агент следует своему контракту самостоятельно.

## Общий принцип

Не вызывать `Skill()` только потому что это "официальный способ". Если задача решается прямым `Edit`/`Read`/`Bash` — использовать инструменты напрямую. Скиллы уместны при неопределённости или нестандартных задачах.
