# Стиль кода — общие правила

- Максимальная длина строки: 120 символов
- Предпочитать именованные экспорты перед default
- Не оставлять закомментированный код — удалять или оформлять в тикет
- Не добавлять TODO без ссылки на тикет (например, `TODO(TICKET-123)`)
- Не добавлять docstrings, комментарии или аннотации типов к коду, который не менялся
- Library-first подход: перед написанием >20 строк новой логики искать готовую библиотеку (WebSearch → Context7 → использовать, если покрытие >70% и активно поддерживается)
- Явное лучше неявного: избегать магических чисел, непонятных сокращений, скрытых побочных эффектов
- Единая ответственность: каждая функция/модуль делает одну вещь хорошо
- DRY в разумных пределах — три похожих строки лучше преждевременной абстракции
- Именование: описательное, однозначное; избегать однобуквенных переменных вне циклов/лямбд
- Импорты: группировать по stdlib → third-party → локальные; сортировать алфавитно внутри групп
- Никаких неиспользуемых импортов, переменных или мёртвого кода
