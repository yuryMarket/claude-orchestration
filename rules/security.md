# Правила безопасности

## Секреты
- Никогда не хардкодить секреты, токены, пароли или API-ключи в коде
- Использовать переменные окружения или менеджеры секретов (Vault, AWS Secrets Manager, SOPS)
- Никогда не коммитить `.env`, `credentials.json`, приватные ключи, `*.pem`, `*.key`
- Проактивно добавлять чувствительные паттерны в `.gitignore`

## Валидация входных данных
- Валидировать все входные данные на границах системы (пользовательский ввод, внешние API, вебхуки)
- Использовать allowlist вместо denylist для валидации
- Экранировать вывод для предотвращения XSS

## OWASP Top 10
- SQL-инъекции: использовать параметризованные запросы / ORM, никогда конкатенацию строк
- Инъекция команд: избегать shell=True, использовать subprocess со списком аргументов
- XSS: экранировать вывод, использовать авто-экранирование фреймворка (React JSX, Jinja2 autoescape)
- CSRF: обеспечить CSRF-токены для запросов, изменяющих состояние
- Path traversal: валидировать и нормализовать пути к файлам

## Зависимости
- Сканировать зависимости: `npm audit`, `pip-audit`, `safety`, `trivy fs`
- Фиксировать версии зависимостей в lockfile
- Проверять новые зависимости перед добавлением (статус поддержки, лицензия, размер)

## Инфраструктура
- Принцип минимальных привилегий для IAM-ролей и сервисных аккаунтов
- Сканировать образы контейнеров: `trivy image`, `grype`
- Включить аудит-логирование для production-систем
- TLS везде — никакого plaintext HTTP в production
